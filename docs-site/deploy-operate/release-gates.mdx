---
title: "Release Gates"
description: "Checks to run before shipping web app or self-hosting changes."
---

## Programmatic checks

Run the checks that match the change:

```bash
npm run typecheck
npm run lint
npm run build
npm run check:config
npm run docs:check
npm run docs:check:self-hosting
npm run check:phase6
```

For focused code edits, targeted ESLint on changed files is acceptable when full lint is blocked by unrelated generated or mobile issues.

## Manual web smoke

Verify:

- App shell loads after sign-in.
- `/api/v1/bootstrap` returns expected user, entitlements, settings, model catalog, and navigation metadata.
- Chat can create a conversation and stream an answer.
- Files can upload, search, open, and delete.
- Memory and knowledge pages render without storage crashes.
- Billing account pages behave correctly for the active billing provider.
- Disabled capabilities hide UI and block server routes.

## Convex gate

If `convex/` changed:

```bash
npm run convex:push:all
```

Then rerun the web smoke against the intended deployment.
