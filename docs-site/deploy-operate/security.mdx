## Route validation

Sensitive `/api/v1` routes should use centralized boundary validation and app API auth helpers. Do not rely on browser-detection headers as an authorization boundary.

## Service auth

Internal server-to-server calls use short-lived service auth. Keep `INTERNAL_SERVICE_AUTH_SECRET` on the app runtime only. Convex needs `INTERNAL_API_SECRET`, matched to the app environment.

## Object access

Uploaded files and generated outputs must remain private and owner-scoped. Browser clients should use short-lived, validated access URLs rather than direct bucket credentials.

## Logging

Do not log:

- Access tokens.
- Session cookies.
- Provider API keys.
- Stripe secrets or webhook signatures.
- Customer document contents unless explicitly part of a safe debugging flow.

Use existing safe logging helpers when adding new server logs.
