---
title: "Self-Hosting"
description: "Run Overlay Web outside the hosted SaaS environment."
---

Overlay Web is a Next.js app plus Convex. The current architecture assumes Convex remains a reachable deployment while the web process can run on any Node-capable host.

## Basic Node deployment

```bash
npm ci
npm run build
OVERLAY_CONFIG_FILE=overlay.config.json npm run start
```

Set `NEXT_PUBLIC_APP_URL`, `NEXT_PUBLIC_CONVEX_URL`, and `INTERNAL_API_SECRET` in the app runtime. Keep secrets in the host secret manager where possible.

## Disable billing

Use `billing.provider="none"` and `capabilities.billing=false`. The billing UI is hidden and billing routes are blocked.

## Disable WorkOS

Use `auth.provider="none"` only for local or controlled self-hosted environments. For enterprise production, prefer OIDC or Keycloak.

## Storage

For on-prem or local deployments, use S3-compatible storage or MinIO. Pair MinIO with path-style access.

## Validation

Run:

```bash
OVERLAY_CONFIG_FILE=overlay.config.json npm run check:config
npm run docs:check:self-hosting
npm run check:phase6
```

The self-hosting examples under `docs/config` are validated by the docs check.
