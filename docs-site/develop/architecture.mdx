Overlay Web is a Next.js app backed by Convex and server-mediated providers for auth, billing, storage, AI, and tools.

## Request flow

1. Browser or client calls `src/app/api/v1/**`.
2. The route wrapper delegates to `handleBffRoute`.
3. `src/server/app-api/v1/**` validates auth, parses input, and coordinates domain services.
4. `src/server/**` talks to Convex, Stripe, WorkOS, storage, model providers, or tools.
5. Shared contracts in `src/shared` and `packages/@overlay/*` keep route callers typed.

## API boundary

`src/shared/schemas/api-boundary.ts` is the route validation and documentation source of truth. It defines the stable web API inventory and request/query/form schemas for `/api/v1`.

When adding or changing a public web API route:

- Add or update the schema in `src/shared/schemas`.
- Add the boundary definition in `api-boundary.ts`.
- Add or update the typed wrapper in `@overlay/api-client`.
- Run `npm run docs:check`.

## Feature modules

`src/features` owns user-facing web feature containers. Feature modules should use shared clients and contracts rather than reaching directly across server or provider boundaries.

## Shared code

`src/shared` must remain isomorphic:

- No Node builtins.
- No `src/server` imports.
- No ad-hoc `process.env`; use public env helpers.
- No client-only React modules.

Verify with:

```bash
npm run check:shared-isomorphic
```
