---
title: "Self-Hosting Obligations"
description: "Operational and branding obligations for self-hosted Overlay Web deployments."
---

Self-hosted deployments are single-customer deployments unless a future multi-tenant architecture explicitly enables shared-tenant isolation.

Before distributing a modified build:

- Review AGPL obligations.
- Replace branding where required.
- Keep secrets out of committed config.
- Validate self-hosting examples and runtime config.
- Run release gates for the target deployment profile.

Recommended checks:

```bash
npm run license:check
npm run docs:check:self-hosting
npm run check:phase6
```
