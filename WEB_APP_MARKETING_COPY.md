# Overlay Web App: Comprehensive Marketing Copy

## Overlay, for people who want one workspace that can think, remember, create, and act

Overlay is an AI workspace built for real work, not one-off prompts. Inside the web app, chat, notes, memories, files, projects, outputs, integrations, and voice all live in one connected system. Instead of scattering context across tabs, tools, and disconnected assistants, Overlay gives you a single environment where every conversation can draw on your knowledge, every project can keep its own context, every generated asset has a home, and every action can move from idea to execution without switching platforms.

It is a web product designed to feel like an operating layer for modern work: part copilot, part knowledge system, part creative studio, part automation surface, part remote execution environment.

## The core promise

Overlay turns a browser tab into a persistent AI workspace that can:

- answer questions with access to your saved context
- take action with tools, integrations, and browser automation
- generate text, images, and video in the same interface
- organize conversations, notes, files, and memories around projects
- capture voice and turn speech into usable work
- connect to external apps

## A unified AI workspace, not a collection of isolated features

Most AI products force a tradeoff: chat is separate from documents, documents are separate from memory, memory is separate from tools, tools are separate from automation, and automation is separate from your actual work context.

Overlay’s web app is built the opposite way.

Your chats can pull from memories and indexed knowledge. Your notes and files can be attached to conversations. Your projects can scope conversations, notes, and knowledge to a specific initiative. Your generated images and videos are stored as outputs instead of disappearing into chat history. Your integrations are available when action is needed.

The result is a product that behaves less like “chat with AI” and more like “operate from one intelligent workspace.”

## What users can do in Overlay today

### 1. Chat with top models across providers

Overlay gives users access to a multi-provider model layer inside one interface. The web app supports models from OpenAI, Anthropic, Google, Groq, xAI, and OpenRouter, with capability-aware model metadata for reasoning, vision, search, and cost tier.

Users can choose the right model for the job instead of committing to one vendor:

- premium reasoning models for complex work
- lighter, faster models for everyday use
- vision-capable models for image-aware tasks
- auto-routed free usage on the free tier

This lets Overlay position intelligence as a configurable layer rather than a fixed backend choice.

### 2. Switch between Ask and Act

Overlay’s chat isn’t limited to answer generation. The web app supports two distinct operating modes:

- `Ask`: for analysis, explanation, synthesis, drafting, and retrieval-backed responses
- `Act`: for tool use, execution, external actions, and multi-step workflows

That distinction matters. It tells users when they are asking for intelligence and when they are asking for execution. In practice, it makes the product feel more trustworthy and more capable, because Overlay can both think and do.

### 3. Bring your own context into every conversation

Overlay is built around persistent context. Conversations can draw from:

- saved memories
- indexed files
- knowledge chunks
- project-scoped context
- attached documents
- prior messages in the same thread

This means users do not need to keep re-explaining themselves. Overlay can search across the user’s knowledge base, retrieve relevant information, and ground responses in material the user already saved inside the workspace.

### 4. Save memory that compounds over time

Overlay includes a dedicated memories system, not just transient chat context. Users can save durable facts, preferences, and recurring context, then revisit, edit, search, and delete that memory later.

That turns the product from a session-based assistant into a workspace that gets better as it learns:

- personal preferences
- standing instructions
- recurring work context
- important facts worth preserving

Users can also manage memories directly in their own view rather than treating memory as a hidden system behavior.

### 5. Build and maintain a living knowledge base

Overlay’s knowledge layer combines files and memories into a searchable knowledge system. Inside the web app, users can:

- upload files
- create folders
- edit supported file types directly
- browse a file tree
- ingest documents into searchable knowledge
- search knowledge across memories and files

This makes Overlay useful for research, internal documentation, personal knowledge management, and project-specific context storage. Instead of pasting reference material into each new prompt, users can keep their source material in the workspace and let Overlay retrieve what matters when needed.

### 6. Write and edit notes in a real notebook environment

Overlay includes a built-in notebook editor, giving users a dedicated writing space instead of forcing everything through chat.

The notes experience supports:

- rich text editing
- headings
- lists
- ordered lists
- quotes
- code blocks
- markdown-style formatting
- slash commands
- autosave
- note tags
- project-linked notes

This makes Overlay suitable for drafting, outlining, structured writing, idea capture, meeting notes, research summaries, and collaborative AI-assisted editing workflows.

### 7. Organize work by project

Overlay is not just a personal scratchpad. It includes project organization as a first-class concept.

Users can group work into projects and keep related assets together:

- project-specific chats
- project-specific notes
- project-specific knowledge and files
- project-specific skills and context

That gives the web app a clear structure for people running multiple initiatives at once, whether those are client accounts, product launches, research tracks, writing projects, or internal teams.

### 8. Generate images without leaving the workspace

Overlay’s web app includes native image generation. Users can create images directly from prompts, choose from multiple image models, and store the result as a persistent output.

The image workflow supports:

- multiple generation models
- aspect ratio selection
- prompt-based generation
- image editing or follow-up generation from an existing image
- saved output records
- download-ready results

This turns Overlay into a creative production surface, not just a writing assistant.

### 9. Generate video in the same flow

Overlay also supports AI video generation. Users can prompt videos from chat, run asynchronous video jobs, and track completed outputs inside a dedicated outputs area.

The video workflow includes:

- multiple video model options
- duration-aware generation
- aspect ratio selection
- asynchronous job handling
- saved output history
- downloadable completed assets

For users creating campaigns, concepts, motion explorations, demo assets, or social content, this expands Overlay from text-and-image generation into multimodal content production.

### 10. Keep every generated asset in an Outputs library

Generated media does not get buried in the chat stream. Overlay includes an Outputs experience where users can browse and filter created assets across image and video.

The outputs system gives users:

- a persistent gallery of generated work
- status tracking for pending, completed, and failed generations
- prompt history
- model metadata
- details panels
- downloads
- quick review of image and video assets

That gives creative generation a real post-generation workflow instead of reducing it to “copy result before you lose it.”

### 11. Use voice as an input layer

Overlay’s web app includes a voice recorder that turns speech into usable text. Users can record audio, transcribe it with AI, and route the result into different destinations.

Voice workflows support:

- browser-based microphone capture
- AI transcription
- copy to clipboard
- save as note
- send into chat workflows

This makes Overlay useful for dictation, quick capture, voice notes, idea dumping, and low-friction hands-free input.

### 12. Connect external tools and services

Overlay includes an integrations layer backed by Composio. Users can browse, search, connect, and disconnect external integrations from inside the web app.

The current product surface explicitly includes integrations such as:

- Gmail
- Google Calendar
- Google Sheets
- Google Drive
- Notion
- Outlook
- X
- Asana
- LinkedIn

And the integration picker is designed to search beyond a short hardcoded list, making the integration story extensible rather than static.

This matters because it lets Overlay move from pure generation into actual operations, whether that means drafting emails, working with calendars, reading files, or triggering downstream tasks in connected systems.

### 13. Browse the live web through AI browser tasks

Overlay’s web app can launch remote browser tasks using a browser automation layer. This gives users a way to request live web work in natural language instead of only receiving static answers.

That includes workflows such as:

- browsing websites for fresh information
- inspecting web pages directly
- running interactive browser tasks
- keeping browser sessions alive for follow-up actions
- optionally routing through regional proxies

This expands Overlay beyond retrieval and into real web execution.

### 14. Maintain long-running context across devices and sessions

Because Overlay’s web app combines conversations, notes, memories, outputs, and project context in one account-based system, users get continuity over time. A conversation does not need to start from zero. A generated asset can remain attached to the broader workspace. A note can later become knowledge. A memory can later shape a future answer.

That continuity is a major product differentiator.

## Why Overlay feels different from a normal AI chat app

Overlay is not selling a better textbox. It is selling a more complete operating model for AI-native work.

### Chat is only one surface

In Overlay, chat sits alongside:

- notes
- memories
- knowledge
- projects
- outputs
- integrations
- voice

That gives the product a broader value story than “ask a model a question.”

### Context is persistent by design

Overlay is built so that useful information can be stored, organized, retrieved, and reused. That makes it suitable for recurring work, not just isolated prompts.

### Action is built in

With Ask and Act modes, browser tasks, and integrations, Overlay is designed for execution as much as ideation.

### Creative production is native

Image and video generation are not bolted-on demos. They are integrated workflows with model choice, output persistence, and a dedicated outputs library.

### The product scales from solo thinking to operational work

A user can start with one quick transcription, move into a note, ask a model to refine it, attach project files, generate supporting visuals, save lasting memory, and then hand a web or computer task off for execution. That entire chain can happen inside one product.
