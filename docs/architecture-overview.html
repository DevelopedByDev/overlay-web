Overlay Web — Architectural Overview

A deep dive into the codebase structure of Overlay's web application: an open-source AI workspace built on Next.js, Convex, WorkOS, Stripe, and Cloudflare.

Scope: Web only (overlay-landing) · Desktop, Mobile, Chrome excluded

1 Executive Summary

Overlay is an open-source AI workspace unifying chat, voice notes, browser tasks, agents, automations, knowledge indexing, and content generation. The web app is built on Next.js 15 (App Router) with React 18, styled with Tailwind CSS, backed by Convex as the real-time database, with WorkOS for auth, Stripe for billing, and Cloudflare R2 for object storage.

Structured as a monorepo with npm workspaces, the codebase contains 16+ internal packages that enforce clean architectural boundaries. Every external dependency is behind a swappable provider interface, enabling self-hosting.

4 Source Code Layering

A strict 4-layer architecture enforced by ESLint boundary rules (scripts/eslint-boundary-rules.mjs):

src/app/
Next.js App Router — routes, layouts, page entry points
src/features/
Feature-domain UI — chat, files, billing, settings, knowledge, automations
src/components/
Shared presentational — sidebar, providers, mentions
src/server/
Server-only — BFF API, services, providers, bootstrap (imports server-only)
src/shared/
Isomorphic — types, schemas, model data, config (no Node builtins, no process.env)
Key rules:
  • src/shared/ — No Node builtins, no process.env (use @/shared/env/public-env), no @/server/*. Verified by npm run check:shared-isomorphic.
  • src/server/ — Imports server-only. Never imported by client code or convex/.
  • convex/ — May import @/shared/* but never @/server/*.
  • src/components/ — May not import from src/features/ (enforced by ESLint).

Auth System

Auth Flow:
  1. User signs in via WorkOS AuthKit → overlay_session httpOnly cookie set
  2. Cookie contains signed payload with WorkOS access token (JWT)
  3. Convex verifies JWT via JWKS + issuer/audience checks
  4. Client fetches Convex token from /api/auth/convex-token (refreshed every 4 min)
  5. API routes support cookie auth (browser) + bearer token auth (native clients)

Billing System

  • Tiered subscriptions: free, pro, max
  • Usage-based credits: creditsUsed accumulator (cents, fractional)
  • Budget reservations: pre-reserve credits before LLM calls, finalize after
  • Daytona sandbox metering: per-second billing for code execution
  • Stripe webhook dedup via processedWebhookEvents table

12 Cloudflare Worker — Chat Stream Relay

A Cloudflare Worker with Durable Objects handles real-time chat streaming:

workers/chat-stream/ ├── src/ │ └── index.ts # ChatStreamDurableObject (30K LOC) └── wrangler.jsonc # Routes: www.getoverlay.io/api/chat-stream/*
  • Durable Object (SQLite-backed) for per-stream state
  • SSE streaming relay between Next.js BFF and browser
  • Dev: Next.js rewrites /api/chat-stream/* to http://127.0.0.1:8787
  • nodejs_compat flag enabled

13 Middleware & Security

CSP (Content Security Policy)

  • Dynamically built from env vars (Convex URLs, R2 origins, PostHog, Sentry)
  • Report-only in development, enforcing in production
  • WebSocket origins for Convex real-time
  • Reports to /api/security/csp-report

Route Protection

TypeRoutes
Protected/account, /api/entitlements, /api/convex
Public/, /auth, /api/auth, /api/security, /api/webhooks
  • API routes: bearer token auth (HMAC + replay protection via serviceAuthReplayNonces)
  • Browser routes: redirect to /auth/sign-in if no session
  • Session cookie signature verified via hasValidSessionCookieSignature()

Security Headers (in next.config.ts)

  • X-Content-Type-Options: nosniff
  • X-Frame-Options: SAMEORIGIN
  • Referrer-Policy: strict-origin-when-cross-origin
  • Strict-Transport-Security (production only)
  • HTML: no-cache (prevents stale CSS bundle mismatches)

15 Data Flow Architecture

High-Level Architecture

Browser (React 18)
↓ ↑
Next.js 15 (App Router) BFF API (src/server)
↓ ↑
Convex (Real-time DB) CF Worker (Stream)
↓ ↑
WorkOS (Auth) Stripe (Billing) R2 / S3 (Storage) OpenRouter (LLM)

Chat (Ask Mode)

Browser → /api/v1/conversations → BFF validation (Zod) → LLM Gateway → Stream via CF Worker (SSE) → Browser renders → Convex persists message + usage

Chat (Act / Agent Mode)

Browser → /api/v1/conversations/act → Build system prompt → Prepare tool registry → Run agent turn (LLM + tool calls) → Execute tools (sandbox / MCP / Composio) → Stream results → Persist + deduct credits

File Upload

Browser → /api/v1/files (intent) → R2 presigned URL → Direct upload to R2/S3 → Convex files table → Optional: knowledge indexing (vector chunks)

Billing

Stripe Webhook → /api/webhooks/stripe → Dedup check → Convex billing sync → Update tier / credits / period
